==================================================
Very Brief Definitions
==================================================
- SQL Injection: An attacker manipulates SQL queries through unsanitized input to read/modify data they should not access.
- Exposed Secrets: Sensitive values (API keys, tokens, passwords) are hardcoded, logged, or stored insecurely where attackers can obtain them.
- Supply Chain Vulnerability: Risk introduced through third-party dependencies, actions, images, or build artifacts that are untrusted, outdated, or tampered.
- Script Injection (Command Injection): Untrusted input reaches shell/script execution paths, allowing attackers to run unintended commands.

==================================================
Detection Parameters (What We Look For In Code)
==================================================
1) SQL Injection
- Query string built dynamically via concatenation/interpolation/formatting.
  - Examples: `"SELECT ... " + userInput`, f-strings containing user data.
- SQL execution calls where first argument is dynamic.
  - Sinks checked via AST include execute-like DB calls.
- Taint indicators: input originating from request/user-controlled variables flowing into SQL query text.
- Severity signal used: tainted dynamic SQL => higher severity.

2) Exposed Secrets
- Hardcoded credential patterns in text/code:
  - keys like `api_key`, `secret`, `token`, `password` with literal values.
- Workflow/job env variables that look secret-like but are plaintext.
  - Expected safe pattern: `${{ secrets.NAME }}`.
- Secret values echoed/printed in CI steps (`echo/printf` with secrets context).
- Artifact upload paths including secret-like files (`.env`, `.pem`, `id_rsa`, token-like names).

3) Supply Chain
- Unpinned dependency versions (`latest`, `*`, loose semver like `^`/`~`).
- Direct URL/VCS dependencies (`git+`, raw `http(s)` package source).
- Known vulnerable dependency/version pairs below built-in safe thresholds.
- Unpinned third-party GitHub Actions references (not full commit SHA).
- Container image tags not pinned (missing tag or `latest`) in Compose/K8s/workflows.
- Missing SCA/security scan steps in workflow (dependency/container scanning absent).
- Unsafe install/update scripts (remote content piped directly to shell, e.g., `curl|bash`).

4) Script Injection (Command Injection)
- Dangerous execution primitives in code:
  - `os.system`, `subprocess.run/Popen/call/check_output`, `eval`, `exec`.
- Shell execution flags that increase risk:
  - `shell=True` (especially with variable input).
- Taint flow to command arguments:
  - request/user data passed into command strings/args without strict validation.
- Workflow run-step patterns that execute remote scripts or untrusted inputs directly.

Practical review checklist
- If input is dynamic + reaches execution/SQL sink => high-risk candidate.
- If secrets are literal or printed => secret exposure.
- If dependencies/actions/images are unpinned or externally fetched without verification => supply-chain risk.
- If shell/eval/exec is used with variable data => script-injection risk.
